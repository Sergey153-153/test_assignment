Небольшое веб-приложение для приёма и обработки заявок в ремонтную службу.

## Требования
- Docker и Docker Compose  
- Windows / Linux / Mac  
- Современный браузер (Chrome, Firefox, Edge)

## Пользователи (сиды в БД)
| Роль       | Логин       | Пароль |
|------------|-------------|--------|
| Диспетчер  | dispatcher1 | 1234   |
| Мастер     | master1     | 1234   |
| Мастер     | master2     | 1234   |

## Запуск проекта

1. Собрать контейнер:
```bash
docker-compose build

Запустить проект:
docker-compose up

Перейти в браузере по адресу: http://localhost:8080

Функционал

Диспетчер:
Создание заявки (клиент, телефон, адрес, описание) → статус new
Список заявок с фильтром по статусу
Назначение мастера → перевод в assigned
Отмена заявки → перевод в canceled

Мастер:
Список заявок, назначенных на текущего мастера
Взять в работу → перевод assigned → in_progress
Завершить → перевод in_progress → done
Отменить → перевод in_progress → canceled
Безопасное “Взять в работу”
Реализовано атомарное обновление через транзакцию и условие WHERE status='assigned'

Если два мастера одновременно нажмут “Взять в работу”:
Первый запрос успешно меняет статус → in_progress
Второй запрос получает ошибку → 409 Conflict, сообщение “Заявка уже взята”

Тестирование через curl
Откройте два терминала и выполните:
# Терминал 1
curl -X POST http://localhost:8080/index.php \
     -d "action=take_in_progress&requestId=1&masterId=2"

# Терминал 2
curl -X POST http://localhost:8080/index.php \
     -d "action=take_in_progress&requestId=1&masterId=3"

Первый запрос → заявка взята в работу
Второй запрос → ошибка 409, сообщение “Заявка уже взята”
